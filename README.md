# 📁 File Organizer (Graph-Based)
Програма на Java за управление на файлове в подкаталози, базирана на графова структура (дървовиден модел).

# 🧩 Описание
Програмата симулира файлова система, в която:

Има главен каталог (root)
Всички файлове се намират само в подкаталози
Подкаталозите са във формат:
YYYY.MM.DD (година. месец. ден)
Всеки подкаталог съдържа до 20 файла
Всеки файл е във формат:
NAME.EXT (само главни латински букви)

# ⚙️ Функционалности
1. Въвеждане на данни
Брой подкаталози (0–100)
Уникални имена на подкаталози
Файлове за всеки подкаталог (валидирани)
2. Сортиране
Подкаталози → по дата (нарастващо)
Файлове →
по име (A–Z)
при еднакво име → по разширение (Z–A)
3. Проверка за дублирани имена
По въведен подкаталог:
„Да“ → ако има файлове със същото име (без значение от разширението)
„Не“ → ако няма
4. Търсене на файл
По въведено пълно име на файл:
извежда всички подкаталози, които го съдържат

# 🔍 Използван алгоритъм за търсене
Използва се линейно търсене `(Linear Search)`
Използван е този алгоритъм, защото:
1. Данните са малко
макс 100 директории
макс 20 файла във всяка
👉 worst case:
100 * 20 = 2000 проверки

# 🏗️ Архитектура
Проектът използва графова структура:

Graph → съдържа всички подкаталози
DirectoryNode → подкаталог + списък с файлове
FileNode → файл (име + разширение)

# 📂 Структура на проекта
`Main.java

FileNode.java`

# ⚠️ Ограничения
До 100 подкаталога
До 20 файла на подкаталог
Максимална дължина на файл: 15 символа
Само главни букви за файлове

# 💡 Бележки
Използвани са Regex за валидация
Използвани са Comparator за сортиране
Проверка за дублирани имена чрез Set
